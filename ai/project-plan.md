# 项目计划

## 项目目标

建设“大连海 AI SaaS”三端一体项目。项目只复用基础脚手架中成熟的用户组织、登录认证、权限控制和安全审计能力，清理无关的代码生成、定时任务、示例入口和默认品牌痕迹，然后进入海事业务和 AI 能力开发。

## 代码组成

| 模块 | 目录 | 基线分支 | 说明 |
| --- | --- | --- | --- |
| 后端服务 | `backend` | Spring Boot 3 | 登录、用户、部门、岗位、角色、菜单、日志、安全配置和后续业务 API。 |
| Web 管理端 | `admin-web` | Vue 3 + TypeScript | 平台管理后台，用于组织权限、业务管理、AI 配置和运营看板。 |
| 小程序/移动端 | `mobile-app` | uni-app + Vue 3 | 移动端工作台，用于业务提交、进度查看、消息和移动 AI 助手。 |
| AI 文档 | `ai` | 当前仓库 | 项目计划、当前准备任务和产品/技术 spec。 |

## 阶段里程碑

### M0: 仓库统一与基线确认

- 合并后端、Web 管理端、小程序三端代码到一个统一仓库。
- 清理三端原有 Git 元数据，使用当前仓库统一管理。
- 建立 `ai` 项目文档和根目录 `agents.md`。
- 推送到 `Wangdongzhi756/dalian_sea_ai.git`。

### M1: 本地可运行基线

- 重命名三端目录和后端 Maven/Java 包名。
- 移除代码生成、定时任务、接口文档页面、默认演示入口等非核心能力。
- 清理默认品牌、默认站点、默认菜单和初始化数据中的脚手架痕迹。
- 保留并验证登录、验证码、用户、部门、岗位、角色、菜单、数据权限、登录日志、操作日志。
- 后端完成 JDK 17、Maven、MySQL、Redis 本地启动验证。
- Web 管理端完成 Node、npm、接口代理和登录验证。
- 小程序完成 HBuilderX/微信开发者工具导入和接口地址配置。

### M2: 开发计划启动

- 建立业务模块命名空间和菜单分组。
- 设计租户、船东、船舶、船员、证书、业务申请、审核流、消息通知等核心模型。
- 输出数据库设计和 API 列表。
- 实现第一个闭环：移动端提交业务，管理端审核，移动端查看进度。

### M3: AI 能力接入

- 接入统一 AI 服务层，支持模型配置、调用日志、费用统计和权限控制。
- 落地文档识别、智能问答、运营助手或海事知识库等首批 AI 场景。
- 建立提示词、评测样例和安全审计机制。

### M4: SaaS 化与交付

- 增强多租户、套餐、计费、审计、数据隔离能力。
- 建立测试、构建、部署和备份流程。
- 准备演示环境、生产部署方案和交付手册。
