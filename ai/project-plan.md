# 项目计划

## 项目目标

建设“大连海 AI SaaS”三端一体项目，以若依体系作为基础工程能力，逐步扩展为面向海事、船东、运营管理场景的 SaaS 平台。

## 代码组成

| 模块 | 目录 | 基线分支 | 说明 |
| --- | --- | --- | --- |
| 后端服务 | `RuoYi-Vue` | `springboot3` | Spring Boot 3.x 后端、权限、系统管理、代码生成、定时任务等基础能力。 |
| Web 管理端 | `RuoYi-Vue3` | `typescript` | Vue 3 + TypeScript 管理后台，用于平台运营、租户和业务配置。 |
| 小程序/移动端 | `RuoYi-App` | `vue3` | uni-app + Vue 3，用于船东、船员或移动业务端。 |
| AI 文档 | `ai` | 当前仓库 | 项目规划、规格、当前任务和长期计划。 |

## 阶段里程碑

### M0: 仓库统一与基线确认

- 合并后端、Web 管理端、小程序三端代码到一个统一仓库。
- 清理三端原有 Git 元数据，使用当前仓库统一管理。
- 建立 `ai` 项目文档和根目录 `agents.md`。
- 推送到 `Wangdongzhi756/dalian_sea_ai.git`。

### M1: 本地可运行基线

- 后端完成 JDK 17、Maven、MySQL、Redis 本地启动验证。
- Web 管理端完成 Node、pnpm/npm、接口代理和登录验证。
- 小程序完成 HBuilderX/微信开发者工具导入和接口地址配置。
- 整理统一环境变量和启动文档。

### M2: 业务模型落地

- 梳理海事 SaaS 核心角色、租户、船舶、人员、订单、通知等实体。
- 设计数据库表、接口、权限菜单和前端页面。
- 建立首个业务闭环：资料维护、业务提交、后台审核、移动端查看。

### M3: AI 能力接入

- 接入统一 AI 服务层，支持模型配置、调用日志、费用统计和权限控制。
- 落地文档识别、智能问答、运营助手或海事知识库等首批 AI 场景。
- 建立提示词、评测样例和安全审计机制。

### M4: SaaS 化与交付

- 增强多租户、套餐、计费、审计、数据隔离能力。
- 建立测试、构建、部署和备份流程。
- 准备演示环境、生产部署方案和交付手册。
